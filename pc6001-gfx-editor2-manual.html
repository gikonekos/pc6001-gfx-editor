01

概要

PC-6001 GFX Editor v2 は、NEC PC-6001 の実機VRAM仕様に準拠したブラウザベースのグラフィックエディタです。 実機の SCREEN 1〜4、アトリビュート構造、色セット定義に基づいて設計されています。

ℹ 仕様出典

本エディタの実機仕様は https://000.la.coocan.jp/p6/tech.html#vram を参照しています。

⚠ v2.0 は実機準拠の土台版です

v2.0 は実機モデルの正確な立て直しを優先した初版です。SCREEN 1のキャラクタ表示・セミグラフィックの完全な編集機能・画像変換の品質追い込みなどは v2.1 以降で対応予定です。

◆ v1系との主な違い

v1系の「SCR1/SCR2/SCR3」は実機のSCREEN番号と対応していませんでした。v2系では実機のSCREEN 1〜4をそのまま使用しています。また v1系の「15色」は実機仕様に存在せず、v2系では CSS ビットによる 4色×2セット（またはモード固有の色定義）を採用しています。

03

VRAM構造

PC-6001 の VRAM は先頭 512バイトがアトリビュート、 それ以降がデータ領域という構造を持ちます。

アトリビュート (先頭512B)

アトリビュートは 32×16 = 512 バイトで、画面の 32 列 × 16 行（または 16 ライン間隔の区画）に対応します。 各バイトのビット構成は以下の通りです。

bit名前意味
7#A/G0=テキスト系, 1=グラフィック系
6#A/Sテキスト時: 0=キャラクタ, 1=セミグラフィック
5#INT/EXTキャラクタ時: 0=内部フォント, 1=外部フォント(CGROM)
4-2GM0,GM1,GM2グラフィック解像度選択
1CSSカラーセット選択 (0 or 1)
0INVキャラクタ反転
SCREENモードアトリビュート値
SCREEN 1 (外部フォント)0x24 (CSS=0) / 0x26 (CSS=1)
SCREEN 2 (64×48セミ)0x60 (CSS=0) / 0x62 (CSS=1)
SCREEN 3 (128×192カラー)0x8C (CSS=0) / 0x8E (CSS=1)
SCREEN 4 (256×192モノクロ)0xDC (CSS=0) / 0xDE (CSS=1)
データ領域 (512B以降)

SCREEN 1/2 はキャラコード/セミグラフィックデータ各 512B。SCREEN 3/4 は 32B × 192行 = 6144B のビットマップデータです。

グラフィックモードでは、アトリビュートは 32 列 × 16ライン単位で画面を制御します。横方向には列単位で分割できますが、縦方向は 16 ライン単位でしか分割できません。

利用方法

基本ワークフロー（SCREEN 3 / 4）
  1. 上部タブで SCR3 または SCR4 を選択
  2. CSS ボタンでカラーセット（CSS=0 / CSS=1）を選択
  3. パレットで FG/BG 色を設定（左クリック=FG、右クリック=BG）
  4. ツールを選んで描画
  5. .p6g 保存 で編集状態を保存 / PNG ダウンロード で画像出力 / ASM で実機VRAM形式テキスト出力
画像読み込み（SCREEN 3 / 4 のみ）

「画像読み込み…」で PNG / JPEG 等を取り込めます。現在のモードと CSS の色セットに最近傍変換されます。 SCREEN 1 / 2 への画像読み込みは v2.0 では非対応です。

アトリビュートグリッド

右パネルの「属性グリッド」で、アトリビュートの区切りを可視化できます。 32列の縦線と、グラフィックモードでは 16 ライン単位の横線が表示されます。 ⊞ アトリビュート選択 ツールでキャラクタ/区画をクリックすると、右パネルにアトリビュートのビット情報が表示されます。

07

エクスポート形式

.p6g 保存（編集状態の完全保存）

v2 の .p6g ファイルは version: 2 を含みます。v1系の .p6g (version: 1) とは互換性がありません。 attr[] + data[] + mode + css をすべて保持します。

ASM 出力（実機VRAM形式）

実機VRAMに書き込み可能な形式で出力します。

; PC-6001 VRAM Data — SCREEN 3
; ATTR: 512 bytes (先頭512B) + DATA: 6144 bytes
;
; ── ATTRIBUTE (512 bytes) ──
VRAM_ATTR:
    DB  08CH, 08CH, 08CH, ...  ; row=0 col=0-15
    ...
; ── DATA (6144 bytes) ──
; SCREEN 3 128×192 2bit/dot MSB=左
VRAM_DATA:
    DB  000H, 000H, ...  ; Y=0
PNG エクスポート

実寸（SCR1/2/3: 128px基準、SCR4: 256×192px）の PNG を生成します。 PNG は表示画像の書き出しです。編集状態の保存には .p6g を使用してください。

注意事項・制限

  • v2.0 は実機準拠の土台版です。SCREEN 1 / 2 のキャラクタ・セミグラフィック編集は簡易表示のみです。
  • 画像読み込みは SCREEN 3 / 4 のみ対応。SCREEN 1 / 2 への変換は v2.1 以降で対応予定。
  • PNG 保存は表示画像のエクスポートです。編集状態の保存には .p6g 形式を使用してください。
  • v1系の .p6g ファイル（version: 1）は読み込めません。v2系は version: 2 形式を使用します。
  • ファイルの出力先はブラウザのセキュリティ制約により選択できません。ブラウザの既定ダウンロードフォルダに保存されます。
  • SCR2 / SCR3 / SCR4 を含む全機能のデバッグ・検証は継続中です。

Overview

PC-6001 GFX Editor v2 is a browser-based graphics editor compliant with the actual PC-6001 hardware VRAM specification. It is based on the real SCREEN 1–4 modes, attribute structure, and color set definitions of the PC-6001.

ℹ SPEC REFERENCE

Hardware specifications referenced from https://000.la.coocan.jp/p6/tech.html#vram

⚠ v2.0 IS A FOUNDATION RELEASE

v2.0 prioritizes correct hardware modeling. Full SCREEN 1/2 character editing, advanced image conversion, and quality improvements are planned for v2.1+.

03

VRAM Structure

The PC-6001 VRAM begins with 512 bytes of attribute data, followed by the display data area.

bitNameMeaning
7#A/G0=text mode, 1=graphics mode
6#A/Stext: 0=character, 1=semi-graphic
5#INT/EXTchar: 0=internal font, 1=external font (CGROM)
4-2GM0,GM1,GM2graphics resolution select
1CSScolor set select (0 or 1)
0INVcharacter inversion

In graphics modes, attributes control 32 columns × 16-line bands. Horizontal (column) splitting is possible but vertical splitting is only in 16-line units.

How to Use

  1. Select SCR3 or SCR4 from the tab bar
  2. Choose a color set (CSS=0 / CSS=1) in the right panel
  3. Set FG/BG colors in the palette (left-click=FG, right-click=BG)
  4. Draw with the tools
  5. Save with .p6g 保存, export PNG, or download assembler output
07

Export Formats

.p6g saves complete edit state (version:2, incompatible with v1 files).

ASM output produces hardware VRAM-compatible format: ATTR block (512B) followed by DATA block.

PNG exports the display image only. Use .p6g for session save/restore.

All exported files are saved to your browser's default download folder (browser security restriction).

Notes & Limitations

  • v2.0 is a foundation release. Full SCREEN 1/2 character/semi-graphic editing is planned for v2.1+.
  • Image import is supported for SCREEN 3/4 only.
  • PNG export is a display image export, not an edit state save. Use .p6g for that.
  • v1 .p6g files (version:1) cannot be loaded in v2. v2 uses version:2 format.
  • Files are saved to the browser's default download folder (cannot be changed due to browser security).
  • Full verification of all modes is ongoing.
